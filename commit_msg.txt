Implement Status Logger and fix 0-index status buffer bugs

Features:
- Added a brand new Status Logger UI Tab to the main window for exporting character and status state to CSV files in real-time.
- Included customizable polling intervals (1, 2, 5, 10, 30, 60, 300, 600 seconds) for the Status Logger.
- Included toggles to turn on/off individual columns for the CSV export (Level, HP, Map, Statuses, Character Name, etc).
- Ensured CSV structure stability: disabled columns will now produce empty cells instead of shrinking the spreadsheet to preserve external alignment formulas.

Bug Fixes & Core Enhancements:
- Performance Fix: Refactored `WorkerCore` state polling to use asynchronous `Task.Delay` instead of synchronous `Thread.Sleep`, resolving a major UI freeze bottleneck during background operations.
- Bug Fix (The SM_ENDURE Bug): Corrected a critical 0-index offset bug in memory reading where the status count was mistakenly processed as a valid `EffectStatusIDs.SM_ENDURE` ailment. All status reading loops across `StatePublisher`, `StatusRecovery`, and `DebuffRecovery` now iterate starting from 1 instead of 0.
- Exception Safety: Wrapped `StatePublisher` loops in robust OperationCanceledException try-catch blocks to prevent silent task death.

Documentation:
- Updated AGENTS.md with "Gotcha #10" documenting the `SM_ENDURE` memory index offset issue to preserve this institutional knowledge.
- Expanded AGENTS.md with a new section documenting the memory layout of the client status buffers.
